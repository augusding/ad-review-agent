# Agent Design Document
# 广告素材合规审核 Agent（Ad Creative Compliance Review Agent）

版本：v1.0  
状态：设计阶段  
负责人：待定  
创建时间：2026-03

---

## 1. 业务问题定义

### 1.1 背景

vivo 商业产品部每日接收大量广告主（App/游戏开发商）提交的广告素材，
涵盖 CPD 应用分发、CPC 效果广告、CPM 品牌展示三种计费形态。
素材类型包括：图片、视频、文案（标题/描述/CTA）、落地页。

### 1.2 当前痛点

| 痛点 | 量化描述 |
|------|---------|
| 人工审核量大 | 日均素材提交量 N 条，高峰期积压超 24 小时 |
| 审核标准不统一 | 不同审核员对边界案例判断差异大，申诉率高 |
| 规则维护滞后 | 广告法/平台规则更新后，人工规则传达不及时 |
| 新人上手慢 | 审核规则复杂，新人需 2-4 周才能独立上岗 |

### 1.3 成功指标（上线后 4 周验收）

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 自动通过率 | ≥ 80% | 无需人工介入直接放行的比例 |
| 漏审率（False Negative） | ≤ 5% | 违规素材被放行的比例，**最关键指标** |
| 误拒率（False Positive） | ≤ 10% | 合规素材被错误拒绝的比例 |
| 平均审核时长 | ≤ 30s | 单条素材端到端处理时间 |
| 人工复核队列处理量 | 减少 60% | 相比纯人工基线 |

## 2. Agent 边界定义

### 2.1 Agent 做什么（In Scope）

- 对提交的广告素材进行多维度合规初审
- 输出结构化审核结论（通过/人工复核/拒绝）
- 输出置信度评分（0-1）
- 输出具体违规项清单（违规内容/所违反的条款/严重程度）
- 输出面向广告主的说明文字（指出问题和修改建议）
- 将低置信度案例推送人工复核队列

### 2.2 Agent 不做什么（Out of Scope）

- ❌ 不做最终审核决定（最终决定权在人工审核员）
- ❌ 不操作广告主账户（封号/扣费/赔付）
- ❌ 不处理广告主申诉（申诉流程独立）
- ❌ 不审核广告的商业策略合理性（仅做合规审核）
- ❌ 不对视频内容做逐帧全量审核（采样审核+关键帧策略）

### 2.3 关键约束

- 每条素材处理成本目标：≤ ¥0.05（含 API 调用）
- 不得存储原始素材内容（隐私合规要求）
- 审核日志保留 90 天（供申诉溯源）

## 4. 审核维度与执行优先级

按以下顺序依次执行，发现 `high` 严重度违规且置信度 > 0.9 时可提前终止后续检测：

### 维度 1：文案违禁词与违禁表述（TextViolationChecker）

**优先级：最高，每条素材必跑**

检测范围：
- 广告法明确禁止用语：「最X」「第一」「唯一」「顶级」「国家级」「保证X」等绝对化用语
- 虚假宣传：「下载量超X亿」「好评率99%」等无法核实的数据
- 诱导性表述：「限时免费」（实为付费）、「系统检测到病毒」等恐吓诱导
- 游戏专项：暗示高额回报的买量话术、未成年人引导
- 金融专项：收益承诺、保本保息表述
- 医疗健康专项：疗效保证、替代医疗表述

检测方式：规则库匹配（高速）+ LLM 语义理解（处理变体和隐性违规）

### 维度 2：图片/视频内容安全（ImageContentChecker）

**优先级：高，图片/视频类素材必跑**

检测范围：
- 色情/暴力/恐怖内容
- 虚假界面截图（伪造系统通知、伪造他人 App 界面）
- 诱导性视觉元素（虚假奖品、诱导点击的箭头/按钮模拟）
- 涉政涉敏内容
- 游戏素材：博彩视觉元素、过度血腥战斗画面

检测方式：Claude claude-sonnet-4-20250514 多模态视觉理解（图片）+ 关键帧抽样（视频，首帧/尾帧/随机3帧）

### 维度 3：落地页一致性（LandingPageChecker）

**优先级：中，有落地页 URL 时执行**

检测范围：
- 落地页内容与素材主张一致性（素材说免费，落地页要付费）
- 落地页是否存在独立的违规内容
- App 名称/开发商名称与素材描述一致性
- 落地页加载状态（404/跳转/反爬）

检测方式：httpx 异步抓取 + 内容比对（降级：无法访问则标注"待人工复查"）

### 维度 4：行业资质匹配（QualificationChecker）

**优先级：中，特殊行业必跑**

需要资质核验的行业：
- 游戏：版号（国家新闻出版署接口核验）、游戏分级标注
- 金融：金融牌照（银保监/证监 号码格式校验）
- 医疗健康：医疗器械/药品广告批准文号
- 教育：办学许可证、学科类培训资质

检测方式：规则库匹配 + 官方接口调用（版号）+ 格式校验（牌照号码）

### 维度 5：vivo 平台专项规范（PlatformRuleChecker）

**优先级：低，在其他维度通过后执行**

检测范围：
- 素材尺寸/格式是否符合对应广告位规范
- 品牌视觉规范（不得使用 vivo 品牌元素误导）
- 竞品攻击性内容（直接贬低其他手机品牌）
- 重复投放检测（与历史已拒审素材高度相似）

## 5. 人机协作设计（HITL）

### 5.1 自动决策阈值

| 条件 | 动作 |
|------|------|
| confidence ≥ 0.92 AND verdict = pass | 自动放行，记录日志 |
| confidence ≥ 0.92 AND verdict = reject | 自动拒审，通知广告主（含违规详情） |
| 0.70 ≤ confidence < 0.92 | 推送人工复核队列，附带 Agent 分析结论作为参考 |
| confidence < 0.70 | 强制人工审核，标注"Agent 低置信度" |
| 任意维度检测异常（超时/报错） | 降级到人工队列，不阻塞广告主提交 |

### 5.2 人工复核队列设计

- 人工审核员看到：Agent 结论 + 置信度 + 具体违规证据 + 原始素材
- 审核员只需：确认/推翻 Agent 结论（不需要重新走完整流程）
- 审核结果：自动写回训练数据集，每周更新 Golden Dataset

### 5.3 申诉处理

- 广告主对拒审结论可发起申诉
- 申诉时自动附带：Agent 审核结论 + 违规证据 + 相关法规条文
- 申诉结果不论如何，作为标注数据收录

## 6. 失败模式与降级策略

| 失败场景 | 降级动作 | 用户感知 |
|---------|---------|---------|
| LLM API 超时（>30s） | 进人工队列 | 提示"审核中，预计1小时内完成" |
| 落地页无法访问 | 跳过该维度，标注"落地页待复查" | 不影响其他维度审核结果 |
| 视频文件过大（>500MB） | 仅审核首帧+随机5帧 | 标注"视频采样审核" |
| 图片 URL 无法访问 | 推人工队列 | 提示补充可访问的素材链接 |
| 版号接口不可用 | 跳过版号核验，标注"版号待人工核验" | 不阻塞审核流程 |
| 输出格式校验失败 | 重试1次，仍失败则推人工队列 | 透明，不暴露技术错误 |

## 7. 技术架构决策

### 7.1 框架选型

**决策：原生实现（不使用 LangChain/LangGraph 等框架）**

原因：
- 审核流程为顺序确定性流程，无需复杂状态图
- DeepSeek 原生 SDK 零摩擦，无适配成本
- 原生实现行为完全可预测，调试效率高
- 框架在此场景属于过度设计

### 7.2 模型分工

| 任务 | 模型 | 原因 |
|------|------|------|
| 文案违禁词检测 | DeepSeek Chat | 成本低，中文理解强 |
| 语义违规判断（边界案例） | DeepSeek Chat | 同上 |
| 图片内容安全检测 | Claude claude-sonnet-4-20250514 | 多模态能力强 |
| 视频关键帧分析 | Claude claude-sonnet-4-20250514 | 同上 |
| 落地页内容分析 | DeepSeek Chat | 成本低，文本理解足够 |

### 7.3 关键技术约定

- 所有 LLM 输出必须经过 Pydantic 验证，不接受裸字符串解析
- 所有 Tool 调用超时设置：网络请求 10s，LLM 调用 30s
- 所有失败必须有降级路径，不允许抛出未捕获异常阻塞流程
- 审核日志异步写入，不阻塞主流程

---

## 8. 数据与隐私

- 素材内容（图片/视频二进制）不持久化存储，仅在内存中处理
- 审核日志存储：request_id / verdict / confidence / violations（不含原始素材内容）
- 广告主 ID 脱敏后用于统计分析
- Golden Dataset 中使用已脱敏的历史数据
